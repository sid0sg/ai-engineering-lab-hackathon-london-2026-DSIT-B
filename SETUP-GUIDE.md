# Setup guide

Everything you need to do before the hackathon. Allow 20 to 30 minutes. Complete this at least two days before the event — do not leave it until the morning.

If you get stuck on any step, contact the support channel listed in your invitation email.

---

## 1. AI coding tools

If you do not have an AI coding tool licence, one will be allocated to you on the day. You do not need to arrange this in advance. The event is tool-agnostic: GitHub Copilot, Amazon Kiro, Gemini Code Assist, and others are all equally valid.

If you already have a working AI coding tool, verify it is active before the day:

1. Open your AI tool's chat panel.
2. Type a simple prompt, for example: `Write a function that reverses a string`.
3. Confirm you receive a suggestion.

If your tool is not responding, contact your department's IT team or the AI Engineering Lab support channel before the event. The [AI Engineering Lab repository](https://github.com/govuk-digital-backbone/aiengineeringlab) has resources on setting up and getting the most from AI coding tools.

Each team has an assigned Forward Deployed Engineer (FDE) who can help with tool setup and any installation issues during the early window on the day.

### Verify features beyond autocomplete

If you already have a working tool, try at least two of the following before the event:

- chat or inline Q&A — ask your tool to explain a piece of code you are working on
- test generation — ask your tool to write tests for an existing function
- refactoring — ask your tool to restructure or simplify a block of code
- agent or multi-file editing — if available, ask your tool to make a change across multiple files

This is not a test. It is preparation. Teams that arrive familiar with their tool's capabilities beyond autocomplete tend to get more done on the day.

## 2. Confirm your device

Bring a developer laptop or a personal device capable of installing software. Your device must be able to:

- run your IDE (or access browser-based tools)
- install extensions if needed (check with IT if your device is locked down)
- connect to the venue wifi

If your device is corporate-managed and you cannot install extensions, check whether your tool has a browser-based option. For example, GitHub Copilot is accessible via github.dev in a browser without local installation. Your FDE can help resolve device issues on the day.

---

## 3. Confirm you have a working development environment

You need:

- at least one language runtime installed (Node.js, Python, Java, or whatever you plan to use)
- a terminal you are comfortable using

You do not need a specific project or framework. You will scope and start your prototype on the day. However, having your environment set up means you can start building immediately rather than spending the first hour on tooling.

---

## 4. Read the problem statements

Problem statements are published one week before the event. Read them before you arrive. Come with an initial sense of which challenge interests you. You will finalise your choice with your team on the morning.

You do not need to prepare a solution beforehand. The point is to arrive with enough context to make a quick decision and start building.

---

## 5. Identify open data sources you might use

Depending on which problem you choose, you may need data. Some starting points:

- GOV.UK open data — https://data.gov.uk
- Office for National Statistics — https://www.ons.gov.uk
- local authority open data portals
- synthetic or mock data is acceptable if real data is not available or raises security concerns

You do not need to download anything beforehand. This is about knowing where to look.

---

## 6. Prepare for submission via GitHub (optional)

One person per team submits the team's work at the end of the day. If your device can access github.com, the preferred submission method is a public GitHub repository shared with your FDE. If you cannot access GitHub, send a zip file directly to your FDE by email or Microsoft Teams — this is equally accepted.

If your team is likely to use GitHub for submission, the person who will create the repository should check the following before the day:

1. Confirm you have a GitHub account. If not, create one at https://github.com/signup — it takes about two minutes.
2. Confirm you can access github.com from your device. Some corporate networks block it.
3. Know how to create a new public repository. From your GitHub homepage, select New, give the repository a name, set visibility to Public, and select Create repository.
4. Know how to share the repository URL. It will be in the format `https://github.com/your-username/your-repo-name`. Send this URL to your FDE before 15:00 on the day.

You do not need to create the repository before the event. This step is about making sure you are not doing it for the first time under time pressure.

## Pre-event checklist

- [ ] You have a developer laptop or personal device capable of installing software
- [ ] If you already have an AI coding tool, it is responding to prompts — if you do not have one, a licence will be allocated on the day
- [ ] Your device can install extensions or access browser-based tools
- [ ] At least one language runtime is installed
- [ ] You have read the problem statements (published one week before)
- [ ] If submitting via GitHub: you have a GitHub account and can access github.com from your device
- [ ] You know your venue and how to get there

## Troubleshooting

| Issue | What to do |
|-------|------------|
| AI tool not responding | Check you are signed in and your licence is active — sign out and back in, then restart your IDE |
| Cannot install extensions on a corporate device | Check whether your tool has a browser-based option — contact IT to request an exception if needed |
| No licence | A licence will be allocated to you during the early setup window — your FDE will help you get started |
| Cannot access github.com | Send a zip file directly to your FDE by email or Microsoft Teams — this is equally accepted |
| Not sure which language to use | Use whatever you are most comfortable with — the problem statements do not require a specific language or framework |
